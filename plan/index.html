¿Cuán factible es realizar ATMAN con Claude Code?

Evaluación honesta para un producto que custodia activos cripto, asumiendo un único developer + Claude Code como copiloto, y producción como objetivo.

Alcance Producción
Equipo Solo + Claude Code
Estado MVP demo listo
Resumen en una línea

Llegar a MVP demo o beta sin custodia con Claude Code solo: muy factible. Llegar a custodia real de fondos en mainnet sin equipo ni auditoría externa: no es factible — y no por límite de Claude Code, sino por límite del modelo "solo dev" para un producto que custodia dinero ajeno.

Veredicto

Tres zonas, tres reglas

El proyecto no es un bloque monolítico: cada capa tiene un nivel de riesgo distinto frente al modelo "solo + Claude Code".

🟢
Adelante

UI, dashboard, CRUDs, tests, CI/CD

Next.js + TS + Supabase + shadcn es exactamente el stack donde Claude Code rinde mejor. Pulir el prototipo, agregar tests, observabilidad y endurecer Supabase: semanas, no meses.

🟡
Con cuidado

Lectura on-chain, KYC, notificaciones

Integraciones con Alchemy/Moralis, Sumsub/Persona, Resend/Twilio. Claude Code las hace, pero el threat model y las decisiones de flujo legal las definís vos — antes de tocar mainnet.

🔴
No solo

Smart contracts, custodia, staking, auditoría

Código que mueve dinero no se shipea sin (a) especialista on-chain y (b) auditoría externa. Trail of Bits, OpenZeppelin o Halborn cobran USD 30k–150k. Compliance LATAM requiere abogado por país.

Claude Code

Dónde da palanca, dónde no alcanza

Una herramienta es buena en lo que es buena. Forzarla fuera de su zona se paga caro cuando hay dinero ajeno en juego.

Dónde da palanca real

  • Velocidad en el stack ya elegidoLlevar el prototipo a beta robusta con tests, error states, accesibilidad e i18n end-to-end es semanas, no meses.
  • Reducción de la deuda actualVitest + Playwright + GitHub Actions + Sentry + linter estricto: días de trabajo guiado.
  • Páginas y formulariosEl patrón ya está. Replicar y extender es trivial cuando el sistema de tipos es sólido.
  • Documentación internaADRs, runbooks, arquitectura — Claude Code los redacta a partir del código real.
  • Migraciones Supabase y RLS policiesBien guiado, escribe SQL y políticas correctas. Revisalas igual.

Dónde no alcanza solo

  • Código que mueve dineroSmart contracts, firma de transacciones, manejo de claves. Sin (a) especialista en seguridad y (b) auditoría externa, shipear a mainnet es jugarse el producto entero.
  • Decisiones de threat model¿Qué pasa si Supabase es comprometido? ¿Si un heredero coludde con un atacante? ¿Nation-state? Estas decisiones las tomás vos.
  • Decisiones legales"Transferencia automática sin abogados" choca con derecho sucesorio real de cada país LATAM. No se programa: se negocia.
  • Operación humanaSi un heredero llama un domingo a las 3 AM, Claude Code no atiende.
Plan por fases

Cómo escalar de demo a producción

Tres fases con factibilidad decreciente para "solo + Claude Code". El salto a Fase 3 implica equipo y capital — no es una decisión técnica.

1
100% factible solo

Demo investor-ready

Duración: 4–6 semanas
  • Pulir prototipo: i18n, error states, loading states, mobile, accesibilidad
  • Vitest + Playwright + GitHub Actions + Sentry
  • Endurecer Supabase: RLS policies, migrations versionadas, backups
  • Páginas legales (Terms, Privacy, Disclaimer cripto)
Outcome Producto demoable a inversores y a early users como "vault de instrucciones cifradas, sin custodia".
2
Factible con disciplina

Beta sin custodia de fondos

Duración: 2–4 meses
  • Lectura on-chain (Alchemy / Moralis) para balances reales
  • Notificaciones reales (Resend + Twilio)
  • Vault como almacenamiento cifrado de instrucciones, no de claves
  • El usuario sigue siendo dueño de sus seeds — ATMAN entrega instrucciones cuando se cumple la condición
Outcome Producto monetizable (Premium $99) sin tocar fondos. Evita 80% del riesgo regulatorio y de seguridad. Defendible legalmente.
3
No factible solo

Custodia + smart contracts

Duración: depende del equipo
  • Levantar capital o socio técnico
  • Auditoría externa antes de mainnet (no negociable)
  • Smart contracts en Solidity con especialista in-house o consultor
  • On-call 24/7 + incident response
Outcome Producto del marketing actual: Premium $99 con staking automático y multisig. Solo posible con equipo + auditoría.
Recomendación

Si seguís solo, no pases de Fase 2.

El plan "Premium one-time $99 con staking automático y multisig" del marketing actual no es un producto que un solo dev deba shipear sin auditoría externa. La buena noticia: una versión sin custodia (entregar instrucciones cifradas a herederos cuando se cumple la condición) cubre el 80% de la propuesta de valor real, es legalmente más defendible, y es 100% construible con Claude Code en pocos meses.

Empezá por endurecer lo que ya tenés. Conseguí early users con Fase 2. Si la tracción justifica Fase 3, levantás capital o socio técnico — no antes.
